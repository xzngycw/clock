# 《认识时间》MVP 开发规格说明（spec）

## 1. 文档目标

本文档用于指导当前代码库的产品化迭代开发。
本期目标是把现有“演示型互动应用”升级为“可持续迭代的儿童时间启蒙 MVP”。

**本期明确不做：**
- 付费/收银/注册码
- 多账号体系
- 教师端/机构端
- 云端同步

---

## 2. 本期产品目标

### 2.1 用户目标
- 孩子可以按课程路径学习“认识时间”
- 孩子可以继续使用现有模拟时钟、数字时钟、练习功能
- 家长可以快速看到学习进度、正确率、错题与建议

### 2.2 业务目标
- 将首页从“功能入口页”升级为“学习入口页”
- 建立课程、练习、错题、家长摘要的基础闭环
- 让现有模块具备后续继续产品化的结构基础

---

## 3. MVP 范围

本期必须完成以下模块：

1. **首页重构**
2. **课程数据与课程列表**
3. **课程详情页**
4. **学习进度记录**
5. **练习结果沉淀到进度系统**
6. **错题记录与错题页**
7. **家长摘要页**
8. **路由与导航补全**
9. **基础验证（build 通过，关键流程可运行）**

---

## 4. 信息架构

```text
首页 Home
├─ 今日学习
├─ 课程地图 / 课程列表
├─ 家长摘要卡
└─ 快捷入口

课程 Lessons
├─ 课程列表页
└─ 课程详情页 Lesson Detail

学习模块 Existing Learning Modules
├─ 认识时间 LearnTime
├─ 模拟时钟 AnalogClock
└─ 数字时钟 DigitalClock

练习 Practice
├─ 难度选择
├─ 答题过程
└─ 结果页

学习沉淀
├─ 错题本 Wrong Questions
└─ 家长摘要 Parent Summary
```

---

## 5. 功能拆解

## 5.1 首页（Home）

### 目标
让用户在首页直接理解“今天学什么、从哪里开始、学到哪里了”。

### 必做内容
1. Hero 区
   - 产品名称
   - 一句价值说明
   - “开始今日学习”按钮

2. 今日学习卡
   - 展示推荐课程
   - 展示当前进度（已完成 / 总课程）
   - 提供“继续学习”按钮

3. 课程地图/课程入口
   - 展示 6 节 MVP 课程
   - 显示每节课状态：未开始 / 学习中 / 已完成

4. 家长摘要卡
   - 学习天数
   - 累计练习时长
   - 正确率
   - 错题数
   - 进入家长摘要页按钮

5. 快捷入口
   - 进入练习
   - 进入错题本
   - 进入现有模拟时钟
   - 进入现有数字时钟

### 验收标准
- 首页首屏能看懂“今天做什么”
- 用户 1~2 次点击可以进入课程或练习
- 能看到基础学习数据

---

## 5.2 课程系统（Lessons）

### 课程范围（MVP 6 节）
1. 认识钟面
2. 认识时针/分针/秒针
3. 认识整点
4. 认识半点
5. 数字时钟与模拟时钟对应
6. 生活中的时间

### 必做内容
1. 课程数据模型
   - 课程 ID
   - 标题
   - 描述
   - 学习目标
   - 预计时长
   - 难度
   - 关联学习页面/目标入口
   - 课程步骤列表

2. 课程列表页
   - 展示全部课程
   - 显示课程状态
   - 支持进入课程详情页

3. 课程详情页
   - 展示课程介绍
   - 展示本节课学习目标
   - 展示学习步骤
   - 提供“开始学习”按钮
   - 提供“标记完成”按钮
   - 提供跳转到关联学习页的能力

4. 课程状态
   - 未开始
   - 学习中
   - 已完成

### 课程步骤结构（统一）
每节课至少包含以下步骤说明：
1. 概念引入
2. 演示理解
3. 互动操作
4. 小练习
5. 完成反馈

### 验收标准
- 能进入课程列表页
- 能打开课程详情页
- 能标记课程完成
- 完成状态会反映到首页和课程列表

---

## 5.3 学习进度系统（Progress）

### 目标
沉淀孩子的学习过程，而不是只保留临时页面状态。

### 必做内容
1. 记录课程完成情况
2. 记录最近学习时间
3. 记录练习总题数 / 正确题数 / 正确率
4. 记录累计练习时长
5. 记录错题列表
6. 生成基础家长摘要数据

### 本期数据来源
- 课程完成：课程页主动标记
- 练习统计：Practice 完成后写入
- 错题：Practice 提交答案时记录

### 验收标准
- 刷新页面后进度仍保留
- 首页/家长页可以读取并展示进度

---

## 5.4 练习系统增强（Practice）

### 目标
保留现有练习功能，同时让练习结果进入产品闭环。

### 必做内容
1. 保留现有难度练习流程
2. 在答题结果中写入：
   - 练习历史
   - 正确率统计
   - 错题记录
3. 错题记录字段包含：
   - questionId
   - questionType
   - difficulty
   - correctTime
   - userAnswer
   - explanation
   - createdAt
   - source（来自哪种练习）

### 本期不做
- 单独的错题再练完整答题模式
- 智能推荐算法
- 专项训练引擎

### 验收标准
- 一次练习结束后，家长摘要数据更新
- 做错的题可以在错题页看到

---

## 5.5 错题本（Wrong Questions）

### 目标
给家长和孩子一个“看到错误、知道薄弱点”的入口。

### 必做内容
1. 错题列表页
2. 展示字段：
   - 题型
   - 难度
   - 正确答案
   - 用户答案
   - 简要解析
   - 记录时间
3. 支持空状态
4. 支持清空错题

### 本期不做
- 错题重新生成专项试卷
- 错题自动归类推荐

### 验收标准
- 有错题时能正常展示
- 无错题时有清晰空状态
- 可清空错题记录

---

## 5.6 家长摘要页（Parent Summary）

### 目标
提供“值得为这个产品买单”的基础价值感知。

### 必做内容
1. 学习总览
   - 已完成课程数
   - 总课程数
   - 连续学习天数（若当前已有）
   - 累计练习时长

2. 练习表现
   - 总答题数
   - 正确题数
   - 正确率

3. 错题摘要
   - 错题总数
   - 最近错题预览

4. 学习建议
   - 若课程未完成：建议继续课程
   - 若错题较多：建议先复习错题
   - 若正确率高：建议提升难度继续练习

### 验收标准
- 家长摘要页有完整信息区块
- 数据与进度、练习、错题保持一致

---

## 5.7 现有学习页复用策略

### 目标
不推翻现有模块，在现有基础上产品化。

### 复用原则
- `LearnTimePage`：作为“概念学习”入口之一
- `AnalogClockPage`：作为模拟时钟专题学习页
- `DigitalClockPage`：作为数字时钟专题学习页
- `PracticePage`：继续作为练习主入口

### 本期要求
- 首页、课程页必须能跳到这些现有模块
- 不强制重写原有教学交互逻辑
- 若需要，只补最小必要联动能力

---

## 7. 路由要求

本期新增路由：
- `/lessons`：课程列表页
- `/lessons/:lessonId`：课程详情页
- `/parent-summary`：家长摘要页
- `/wrong-questions`：错题页

保留现有路由：
- `/`
- `/learn-time`
- `/analog-clock`
- `/digital-clock`
- `/practice`

---

## 8. 状态管理要求

## 8.1 progressStore 需要负责
- 初始化学习进度
- 课程完成状态更新
- 错题记录增删
- 练习结果沉淀
- 家长摘要数据生成

## 8.2 gameStore 继续负责
- 当前练习会话
- 当前题目
- 答题过程
- 会话结束

### 原则
- `gameStore` 管练习过程
- `progressStore` 管持久化学习沉淀

---

## 9. UI/UX 要求

### 全局要求
- 移动端优先
- 一屏一主任务
- 首屏信息密度清晰
- 儿童友好、按钮大、反馈明确

### 页面要求
1. 首页：学习入口优先
2. 课程页：状态清晰、路径清晰
3. 错题页：信息直观，避免复杂操作
4. 家长页：以数据卡片为主

### 交互要求
- 所有关键按钮要有明确文案
- 空状态要友好
- 状态切换要有反馈

---

## 10. 技术实现要求

### 本期必须保证
- `npm run build` 通过
- 新增模块之间数据联通正确
- 不破坏现有学习/练习主流程
- 本期新增代码优先复用现有 store / utils / layout

### 本期建议
- 以静态课程数据驱动页面
- 避免一次性引入复杂配置后台
- 页面可先用本地 store + 本地持久化支撑

---

## 11. 开发顺序

### Phase A
- 重写 `spec.md`
- 补齐类型 / 常量 / 课程数据

### Phase B
- 补充路由
- 新增课程列表页 / 课程详情页

### Phase C
- 扩展 progressStore
- 接入错题记录与家长摘要

### Phase D
- 重构首页为学习入口页
- 接入课程进度与推荐任务

### Phase E
- 新增错题页 / 家长摘要页
- 联调现有练习页数据写入

### Phase F
- 构建验证
- 回归关键流程

## 13. 本期不做
- 会员付费逻辑
- 注册码验证
- 远端账户体系
- 教师/班级功能
- 个性化推荐算法
- 云端同步与多设备同步

---

## 14. 实施说明

本期开发遵循以下原则：
1. 先把产品结构补齐，再逐步增强内容深度
2. 先打通首页 → 课程 → 练习 → 错题/摘要闭环
3. 现有功能尽量复用，不做无必要推翻重写
4. 付费能力只做未来兼容性预留，不在本期实现
